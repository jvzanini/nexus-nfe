# Fase 20 — 2FA TOTP (spec final)

**Objetivo:** Segundo fator TOTP (Google Authenticator, Authy, 1Password) opcional por usuário. Obrigatório para super_admin após ativação global.

## Escopo IN
- Setup em `/profile/security` (novo sub-tab em /profile):
  - Botão "Ativar 2FA" → gera secret → QR code → usuário confirma com código → salva `twoFactorEnabled=true`, `twoFactorSecret` (criptografado com `ENCRYPTION_KEY`).
  - Backup codes: 10 códigos únicos gerados no ativar, exibidos 1x (download txt).
  - "Desativar 2FA" exige código TOTP válido + senha atual.
- Login flow:
  - Após credenciais OK, se usuário tem 2FA ativo, redirecionar para `/login/2fa` onde digita 6 dígitos.
  - Aceita backup code (10 dígitos hex) uma vez — consome o backup.
- Audit: `user.2fa_enabled`, `user.2fa_disabled`, `user.2fa_login_ok`, `user.2fa_login_failed`.

### Escopo OUT
- WebAuthn/FIDO2 (próxima fase).
- 2FA obrigatório por padrão (opt-in).

## Model changes
```prisma
model User {
  ...
  twoFactorEnabled  Boolean @default(false) @map("two_factor_enabled")
  twoFactorSecret   String? @map("two_factor_secret")   // criptografado
  twoFactorBackup   String? @map("two_factor_backup")   // JSON array criptografado de códigos ainda válidos
  twoFactorVerifiedAt DateTime? @map("two_factor_verified_at")
}
```

## Stack
- `otplib` (package já auditável) — geração de segredo + verificação TOTP.
- `qrcode` para gerar QR dataURL server-side.
- `crypto.randomBytes` para backup codes.
- Criptografia dos secrets via `encrypt/decrypt` de `src/lib/encryption.ts` (já existe, AES-256-GCM).

## Fluxo NextAuth
NextAuth v5 authorize callback atual valida senha. Passo 2FA fica **fora** do callback (evita problemas com JWT). Approach:
1. `signIn` normal. Se `user.twoFactorEnabled`, armazenar um flag `needs2fa=true` no JWT e marcar sessão como "pré-autenticada".
2. Middleware bloqueia rotas protegidas quando `needs2fa=true`, redireciona para `/login/2fa`.
3. `/login/2fa` valida token/backup, chama `signIn("credentials", { user, otp })` OU usa server action que reescreve o JWT via `unstable_update` do NextAuth.

Simplificação: fazer tudo no `authorize`:
- input pode incluir `otp` opcional.
- se user tem 2FA e `otp` não fornecido → lança Error com code `2fa_required`.
- client captura esse erro e exibe step 2FA no próprio `/login`.

## UI
- `/login` — step 2 aparece após step 1 quando necessário.
- `/profile/security` — novo tab em /profile:
  - Card "2FA": status + ativar/desativar.
  - Dialog de setup: QR + secret como texto + input de confirmação + backup codes.
